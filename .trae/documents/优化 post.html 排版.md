## 现状确认
- `post.html` 是唯一的后端模板文件，CSS 为内联样式。
- 页面正文只渲染 `{{.Body}}`，`{{.Title}}` 目前仅用于浏览器 `<title>`，不在页面中展示。

## 设计目标（简约而精美、轻松阅读）
- 控制行宽（阅读舒适的 65–75 字符宽度），提升段落节奏与层级。
- 减少“强装饰”（渐变、重阴影、图片 hover 位移），改成更克制的纸张感与细边框。
- 适配中文排版：取消段落两端对齐与自动断词，避免字间拉伸。
- 提供暗色模式、打印样式与减少动效支持（可访问性）。

## 模板结构调整
- 将正文包裹在更语义化结构中：`<main>` + `<article>`，并新增一个可选的标题区块。
- 默认在页面内展示 `.Title` 作为文章标题（因为当前页面不展示标题），并将正文区域限定为“内容排版容器”。

## CSS 改造方案（仅改 `post.html`）
- 引入 CSS 变量（颜色/间距/圆角/阴影），统一风格并便于后续微调。
- 布局：
  - 使用 `width: min(72ch, 100%)` 控制可读行宽；
  - 使用 `clamp(...)` 做自适应内边距与标题字号；
  - 背景改为轻微中性色，卡片使用细边框 + 轻阴影。
- 排版（markdown 常见元素）：
  - `p` 左对齐、合适行高与段距；
  - `h1-h4` 更简洁的层级（弱化大标题的装饰线条）；
  - 链接使用下划线与 `text-underline-offset`；
  - `blockquote` 改为浅底 + 细左边框；
  - `code/pre` 改为更克制的高对比方案（保持易读与复制友好）；
  - `table` 增加横向滚动容器与更轻的边框样式；
  - `img` 去掉 hover 位移，仅保留适度圆角与间距。
- 响应式：在 768/480 断点下优化字体与留白，避免移动端过大边距。
- 可访问性：为键盘 focus 增加可见的 `:focus-visible` 样式；加入 `prefers-reduced-motion`；加入 `@media print`。
- 暗色模式：使用 `prefers-color-scheme: dark` 覆盖变量，保证对比度。

## 验证方式（你确认计划后我再执行）
- 启动后端并打开任意文章页，检查：标题是否显示、段落是否更舒适、代码块/表格/图片在桌面与移动端是否正常、暗色/打印是否可用。
