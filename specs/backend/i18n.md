# Backend Internationalization

## Overview

The backend uses `gin-contrib/i18n` with `go-i18n/v2` for server-side message localization. Error messages and response strings are translated based on the client's `Accept-Language` header.

## Locale Files

Locale files are stored in `backend/locales/` as TOML files:

```
backend/locales/
├── en.toml              — English (default)
├── zh.toml              — Chinese (legacy)
├── zh-hans.toml         — Chinese Simplified
├── active.en-us.toml    — Active English US
└── active.zh-hans.toml  — Active Chinese Simplified
```

### Format

Each entry is a TOML array of tables with `other` as the translated string:

```toml
["error.invalid_credentials"]
other = "Invalid username or password"

["error.not_found"]
other = "Not Found"
```

## Configuration

i18n is configured in `cmd/server/main.go` during server setup:

```go
r.Use(ginI18n.Localize(ginI18n.WithBundle(&ginI18n.BundleCfg{
    RootPath:         "./locales",
    AcceptLanguage:   []language.Tag{language.English, language.Chinese},
    DefaultLanguage:  language.English,
    UnmarshalFunc:    toml.Unmarshal,
    FormatBundleFile: "toml",
})))
```

Supported languages: English (`en`) and Chinese (`zh`). English is the default.

## Language Detection

Language is detected from the `Accept-Language` HTTP header. If the header is missing or the language is not supported, English is used as the fallback.

## Usage in Code

Handlers use `ginI18n.MustGetMessage` to get translated strings:

```go
message := ginI18n.MustGetMessage(c, &i18n.LocalizeConfig{
    DefaultMessage: &i18n.Message{
        ID:    "auth.logout_success",
        Other: "Logged out successfully",
    },
})
```

The `DefaultMessage` provides a fallback if the message ID is not found in any locale file. Error messages in `pkg/apierr/` follow the same pattern — each error code maps to an i18n message ID.

## Message ID Naming Convention

Message IDs use dot-separated, feature-prefixed names:

- `error.invalid_credentials` — Error messages
- `error.not_found` — Error messages
- `auth.logout_success` — Auth feature
- `health.running` — Health check

All message IDs are lowercase with underscores.
